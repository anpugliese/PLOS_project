Nowadays, the Geographic Information Systems have evolved and developed in a way that technologies are available for many people, facilitating the processing and analysis of geographic data.

The development of this project involved the use of many technologies for the data collection, processing and analysis that, altogether, have allowed us to describe the walkability of “Isola”, a neighbor in Municipio 9, Milan, Italy.

Data Collection

Data collection is the first and one of the most delicate and expensive steps in any GIS project as it involves on-field surveying, a time consuming task.

Data collection was performed directly at the site that was designated to us, by going into each street and filling all the data, together with GPS coordinates, inside the Epicollect5, a mobile application also used and supported by the Center for Genomic Pathogen, University of Oxford and the British Big Data Institute.

The data we collected included: number of lanes, on street parking, buffer, width of sidewalk, curb ramps, lightings, width of bike lanes and shoulders. For each data entry time, GPS coordinates and accuracy were also recorded by Epicollect5.

Data transfer

Data transfer was a simple task that we performed by using Epicollect5 mobile application. Once the field data collection was concluded, it was uploaded to Epicollect5 cloud service and then downloaded as a csv format to our computers. Since the points were collected by each of us using her/his Epicollect5 account, the data transfer consisted of 4 different csv files which, later, had to be merged before processing.

Data processing

This step was certainly the most challenging of the whole process, not only because of the number of actions involved, but also due to the fact that some data cleaning was required, a task that could not be completely automated.

The main steps in our data processing were:

  • 1. Download data
  • 2. Merge data into a unique file
  • 3. Check data consistency
  • 4. Import data to QGIS
  • 5. Display data in QGIS
  • 6. Improve data quality (e.g. manual relocation of points that were misplaced due to low accuracy)

    • Data Analysis

    Data analysis can be seen as the most interesting part of the project, since here we elaborate and visualize the final result of our project. Data analysis was performed using QGIS, a free and open source GIS software developed by Open Source Geospatial Foundation (OSGeo). Using QGIS we were able to process the data located in our Epicollect5 csv file to project it and compute PLOS/EPLOS formulas and, finally, to visualize results on the map.

    Data presentation

    The final step was to expose the results of the analysis through a website. This step required mastering several technologies, including:

  • 1. HTML web page programming
  • 2. CSS website styling
  • 3. Java interfaces
  • 4. Geoserver storage and implementation
  • 5. GeoJason files visualization
  • 6. GitHub files hosting and visualization
  • 7. Map layers

    • After the collection and consolidation of all these technologies we obtained this web GIS, where everything works in a harmonious and efficient way.
