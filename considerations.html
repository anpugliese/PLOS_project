Technologies and data used

Nowadays, the Geographic Information Systems have evolved and developed in a way that technologies are available for many people, in this way they facilitate the processing and analysis of geographic data. The development of this project involves the use of many different technologies for the data collection, processing and analysis, that altogether will help us describe the walkability of a designated area in Milan.

The following are the steps taken to be able to achieve the final result presented through the different sections in this webpage:

1. Collection

2. Transfer

3. Processing

4. Analysis

5. Presentation

Throughout this section we will explain each of these steps and the different tools and technologies used in our project development.

Data Collection

Data collection was the first and one of the most complicated steps in this project. The reason for this difficulty is that data necessarily needed to be collected with specific characteristics which involved: precision, efficiency, low cost and structure. The solution to this was to use a mobile application called EpiCollect5, this application is used and supported by certain institutions such as: Center for Genomic Pathogen, University of Oxford and the British Big Data Institute. As it is described in their website:

“Epicollect5 is a mobile & web application for free and easy data collection. (...)

It provides both the web and mobile application for the generation of forms (questionnaires) and freely hosted project websites for data collection. Projects are created by using the web application at five.epicollect.net, then downloaded to the device to perform the data collection.

Data are collected (including GPS and media) using multiple devices and all data can be viewed on a central server (via map, tables and charts).

Data can be exported in csv and json format.

Epicollect5 is 100% free to use without any limits(*). You can create as many projects as you like and upload as many entries as you like. Epicollect5 is financially supported by the Wellcome Trust Foundation and we implement open source technologies to be able to provide the service for free, forever.”

Data collection was performed directly at the site that was designated by our professors, by going to each one of the streets to be classified with the PLOS/EPLOS methodology and filling all the data inside the Epicollect5 application. This collection also involved registering the GPS coordinates of each piece of data; information that later would be used to locate the information in our map.

Data transfer

Data transfer is a very simple task that could be easily completed, again by using the Epicollect5 mobile application. Once the field data collection was concluded, it was uploaded to the cloud on Epicollect5 servers and afterwards downloaded as a csv format to our computers. Since the points were collected by different users in our field work, the data transfer consisted of 4 different csv files which needed to be merged together and processed.

Data processing

This step of the process was certainly the most challenging of them all, not only because of the amount of steps required to complete it, but due to the fact that data cleaning by nature is demanding since it cannot be completely automated, manual work is always needed. Data cleaning took a lot of time which were divided in the following steps:

1. Download data

2. Merge data into a unique file

3. Arrange data to a usable format (one column for each type of data

4. Manual verification that data is correct

5. Import data to GIS software

6. Display data in GIS software

Improve data quality (relocate points that due to accuracy problems were not located where they should)

Data Analysis

Data analysis can be seen as the most interesting part of the project, since here we visualize the final result of our project. The technology used for the Data Analysis was the GIS software called QGIS. This is an open source software developed by Open Source Geospatial Foundation (OSGeo) and used by a worldwide community. A GIS software is:

“GIS, in full geographic information system, computer system for performing geographical analysis. GIS has four interactive components: an input subsystem for converting into digital form (digitizing) maps and other spatial data; a storage and retrieval subsystem; an analysis subsystem; and an output subsystem for producing maps, tables, and answers to geographic queries. GIS is frequently used by environmental and urban planners, marketing researchers, retail site analysts, water resource specialists, and other professionals whose work relies on maps.”(Britannica 2020)

Using this software we were able to process the data located in our Epicollect5 csv file to project it on a map and apply the PLOS/EPLOS formulas to finally visualize it on our map (presented on the previous section).

Data presentation

The final step in our whole process was to present the data correctly to be visualized by the interested people. This results presentation is done through this website, where several technologies are involved:

1. HTML web page programming

2. CSS website styling

3. Java interfaces

4. Geoserver storage and implementation

5. GeoJason files visualization

6. GitHub files hosting and visualization

7. Map layers

After the collection and consolidation of all these technologies we obtain this project as a result, where everything comes to reality. Map interaction, information presentation and data visualization merge into one global technology assembly that works in a harmonious and efficient way.
